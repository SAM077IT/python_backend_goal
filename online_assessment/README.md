## 🗺️ Assessment structure

```
Online Assessment (CoCubes)
├── Round 1 — Cognitive & Technical (MCQ)
│   ├── Verbal ability
│   ├── Logical reasoning
│   ├── Pseudo-code tracing
│   └── Networking / Cloud basics
│
├── Round 2 — Coding Assessment  ← Primary filter
│   ├── 2–3 problems in 45–60 minutes
│   ├── Python allowed
│   └── Target: 2 fully correct + 1 partial
│
├── Round 3 — Communication Assessment
│   └── Spoken + written English (pass/fail for dev roles)
│
└── Round 4 — Technical + HR Interview (if OA cleared)
    ├── Python deep dive (decorators, OOP, memory mgmt)
    ├── SQL + Pandas/NumPy
    ├── Flask/Django REST API design
    ├── AI/ML awareness
    └── STAR-format behavioral questions
```

## ⚡ Key facts (from Glassdoor reports — May 2026)

- Decorators, OOP, `args`/`kwargs`, memory management are high-frequency topics
- Dictionary manipulation — often asked to show **multiple approaches**
- NumPy and Pandas coding challenges appear in technical rounds
- SQL query writing is expected (joins, GROUP BY, window functions)
- AI/ML conceptual awareness is now explicitly tested
- Sectional cut-offs apply — you must pass **each sub-section** individually
