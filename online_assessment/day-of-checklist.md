# ✅ Day-of Assessment Checklist

> Run through this the **evening before** and again **30 minutes before** your assessment starts.

---

## 🌙 Evening Before

### Environment setup
- [ ] Stable internet connection tested — run a speed test (`fast.com`)
- [ ] Backup connection identified (mobile hotspot, if needed)
- [ ] Laptop/desktop plugged in and charging
- [ ] Browser tested: Chrome is recommended for CoCubes
- [ ] Browser extensions that block scripts are **disabled** (uBlock, NoScript, etc.)
- [ ] Assessment login link confirmed and bookmarked
- [ ] Camera tested — face clearly visible, good lighting
- [ ] Microphone tested (for communication assessment)
- [ ] Quiet, private room identified and arranged

### Stationery
- [ ] Pen and paper ready and **visible to the camera** (allowed, but must be in frame)
- [ ] Scratch paper for working out problems

### Mental prep
- [ ] Review your notes on: decorators, OOP dunder methods, `*args`/`**kwargs`
- [ ] Re-read your 3–4 STAR behavioral stories once
- [ ] **No new topics tonight** — just consolidate
- [ ] Set an alarm for the morning with buffer time
- [ ] Sleep by a reasonable hour — cognitive performance drops significantly with < 7 hours

---

## 🌅 Morning of Assessment

### 30 minutes before start
- [ ] Phone on **silent** mode, notifications off
- [ ] All other browser tabs closed
- [ ] Assessment link open and ready to begin
- [ ] Pen, paper, and water bottle on desk
- [ ] Comfortable temperature in room
- [ ] Take 5 slow breaths — calm focus beats anxious speed

---

## 🖥️ During the Assessment

### Cognitive & Technical MCQ section
- [ ] Don't spend more than 90 seconds on any single MCQ
- [ ] Flag uncertain questions and return — don't get stuck
- [ ] For pseudo-code questions: trace line by line on paper
- [ ] For networking/cloud MCQs: eliminate obviously wrong answers first
- [ ] Remember: sectional cut-offs apply — attempt every sub-section

### Coding section (45–60 minutes)
- [ ] **Read all problems first** before writing any code
- [ ] Start with the problem you find easiest
- [ ] Write a brute-force working solution first — even O(n²) gets partial credit
- [ ] Test with the provided sample input before submitting
- [ ] Test your own edge cases: `[]`, single element, all duplicates, negatives
- [ ] If stuck after 15 min: move to next problem, return if time allows
- [ ] Optimize only after all problems have a working attempt

**Coding time guide:**
```
Problem 1 (easiest) → aim for ≤ 12 minutes
Problem 2 (medium)  → aim for ≤ 20 minutes
Problem 3 (hardest) → remaining time, partial is fine
```

### Communication section
- [ ] Speak in complete sentences — don't trail off
- [ ] Take 3–5 seconds to think before answering; silence is better than filler words
- [ ] Speak clearly and at moderate pace
- [ ] For written tasks: check grammar and sentence structure before submitting

---

## 💡 If Things Go Wrong

| Situation | What to do |
|-----------|------------|
| Internet drops mid-test | Switch to mobile hotspot immediately; most platforms allow resume |
| Platform crashes | Screenshot error, contact Accenture HR / CoCubes support right away |
| Blank on a coding problem | Write pseudocode in comments + brute-force loop — shows thinking |
| Running out of time | Submit partial solutions — partial output scores count |
| Mind goes blank on MCQ | Flag it, move on, return with fresh eyes |

---

## 📌 Key Numbers to Remember

| Metric | Target |
|--------|--------|
| Coding problems to fully solve | 2 of 2–3 |
| Coding problems partial is okay | 1 |
| Time per coding problem | ≤ 20 minutes each |
| MCQ confidence threshold to move on | If unsure after 90s, flag and continue |

---

## 🎯 You're Ready When...

- [ ] Can write a Python decorator from scratch without notes
- [ ] Can explain the difference between shallow and deep copy with an example
- [ ] Can trace through a 10-line pseudo-code snippet and state its output
- [ ] Have solved ≥ 10 coding problems total
- [ ] Have done ≥ 1 full timed mock session
- [ ] Know your 3–4 STAR stories by heart
- [ ] Environment is set up and tested

---

> **Final note:** At 3–4 years of experience, you already know Python well.
> The assessment is testing whether you can apply that knowledge under time pressure
> and communicate it clearly. Stay calm, work systematically, and trust your preparation.

**Good luck! 🚀**
